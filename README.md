## 현재 운영 정책

### 봇 판정

- 봇은 게임 참가자로 취급합니다.
- 봇 죄수가 살아 있으면 기본적으로 LR을 막습니다.
- 봇은 간수 비율 계산에서는 제외합니다.
- 봇 간수는 LR 상대가 될 수 있습니다.

테스트 목적으로 봇 죄수를 무시하고 LR을 열고 싶다면 `BotsBlockLastRequest`를 `false`로 바꿀 수 있습니다. 운영 기본값은 `true`입니다.

### 간수 지시

- `!지시`, `/지시`, `!order`, `/order`로 열 수 있습니다.
- 관리자 권한자가 아니면 CT 간수만 지시할 수 있습니다.
- T 상태에서는 지시할 수 없습니다.
- 살아있음 여부는 보지 않습니다. CS2에서 `PawnIsAlive` 판정이 순간적으로 흔들려 지시가 막히던 문제가 있었기 때문입니다.
- LR 진행 중에는 새 간수 지시를 막습니다.

### 간수 지시 HUD

간수 지시는 중앙 텍스트로 반복 출력됩니다.

```text
[간수 지시]
중앙로 6:30까지 이동
```

CS2의 `PrintToCenterHtml`은 환경에 따라 흰 박스만 표시되거나 깜빡일 수 있어 현재는 일반 `PrintToCenter`를 사용합니다.

### 지시 메뉴

기본 장소 메뉴는 첫 페이지에 4개 장소와 자유 입력을 배치합니다.

```text
1. 중앙
2. 수영장
3. 샤워실
4. 운동장
5. 자유 입력
```

`감방`은 메뉴에서 숨깁니다. 페이지 넘김 실수로 다른 지시를 내리는 상황을 줄이기 위해 자유 입력을 첫 페이지 5번에 둡니다.

### LR

- 살아 있는 LR 차단 대상 죄수가 1명일 때 마지막 죄수로 감지합니다.
- 기본 설정에서는 봇 죄수도 LR 차단 대상입니다.
- 마지막 죄수가 `!lr`을 입력하면 메뉴를 엽니다.
- 자동 감지가 늦어도 `!lr` 입력 시 조건이 맞으면 후보를 복구합니다.
- LR 참가자 외 데미지는 차단합니다.
- 칼전에서 총 데미지는 차단합니다.
- 서버 안정성을 위해 LR 중 반복 무기 제거는 하지 않습니다.
- LR 시작 시에만 무기를 정리하고 지급합니다.

현재 구현된 게임:

- 칼전
- 권총전
- 노스코프전

### 반란자

- 죄수가 간수를 공격하면 반란자로 표시됩니다.
- 반란자는 빨간색 렌더 컬러를 적용합니다.
- 스킨 플러그인이 색을 덮어쓰는 경우를 줄이기 위해 1초마다 반란자 색을 다시 적용합니다.
- 라운드 종료, 팀 변경, 퇴장, 초기화 때 상태를 정리합니다.

### 미니맵

플러그인 로드와 맵 시작 시 설정된 서버 명령을 실행합니다.

기본값:

```text
sv_disable_radar 1
```

CS2 버전이나 서버 설정에 따라 해당 cvar가 동작하지 않을 수 있습니다. 실패 시 서버 로그에 경고만 남깁니다.

## 운영 정책

### LR 제거

LR은 현재 빌드에서 제거했습니다.

CS2 서버 안정성을 우선하기 위해 LR 메뉴, LR 데미지 차단, LR 무기 지급, 줌 차단, 마지막 죄수 자동 감지는 등록하지 않습니다. `Features/LastRequest` 구현도 제거되어 빌드 대상에 포함되지 않습니다.

### 간수 지시

- `!지시`, `/지시`, `!order`, `/order`로 메뉴를 엽니다.
- 관리자 권한자가 아니면 CT 간수만 지시할 수 있습니다.
- T 상태에서는 지시할 수 없습니다.
- 살아있음 여부는 보지 않습니다. CS2에서 `PawnIsAlive` 판정이 순간적으로 흔들려 지시가 막히던 문제가 있었기 때문입니다.
- 새 지시가 나오면 전체 자유시간은 종료됩니다.
- 라운드 중에는 지시가 없어도 중앙 HUD에 `자유시간: 없음`, `추가명령: 없음`, `최근사건: 없음`을 표시합니다.

### `$` 빠른 시간 지시

일반 채팅에서 `$`로 시작하면 CSS식 빠른 시간 지시로 처리합니다.

```text
$6:30 중앙 앉아서
$6 중앙 앉아서
$90 중앙 앉아서
```

위 예시는 각각 `6:30까지 중앙 앉아서`, `6:00까지 중앙 앉아서`, `1:30까지 중앙 앉아서`로 표시됩니다.

시간은 현재 남은 라운드 시간보다 작아야 하며, 기존 간수 지시 쿨다운과 최소 시간 제한을 그대로 사용합니다.

자유 입력 메뉴에서 입력한 내용은 `$`로 시작하더라도 파싱하지 않고 그대로 간수 지시 HUD에 표시합니다.

### `+` 추가명령

일반 채팅에서 `+`로 시작하면 추가명령 줄을 갱신합니다.

```text
+점프 금지
+총 줍지 말기
+
```

빈 `+`는 추가명령을 `없음`으로 되돌립니다.

### `!5` 카운트다운

- `!5`, `/5`, `css_jb countdown`, `css_jb 5`로 5초 카운트다운을 시작합니다.
- 간수 또는 권한 있는 관리자만 사용할 수 있습니다.
- 기존 카운트다운이 있으면 새 카운트다운으로 교체됩니다.
- 모든 사람에게 중앙 메시지와 채팅 메시지를 보냅니다.
- 최근 사건에 카운트다운 시작 기록을 남깁니다.

### 프리킬 시간

- 라운드 시작 시 살아 있는 간수가 3명 이상이었고, 이후 살아 있는 간수가 1명으로 줄어드는 순간 자동으로 시작됩니다.
- 살아 있는 죄수가 1명 이상 있을 때만 시작합니다.
- 전체 자유시간 중에도 프리킬 조건이 충족되면 프리킬 시간이 우선합니다.
- 프리킬 시간이 시작되면 자동 자유시간 타이머와 자유시간 HUD를 종료합니다.
- 프리킬 시간 중에는 간수 지시, 추가명령, 최근사건 대신 중앙 HUD에 `프리킬 시간 입니다!`를 고정 표시합니다.
- 프리킬 시간 중에는 전체 자유시간과 카운트다운을 새로 시작할 수 없습니다.
- 매초 감시하지 않고 라운드 시작, 사망, 팀 변경 같은 이벤트 후에만 한 번 계산합니다.
- 짧은 시간에 사망/팀 변경/퇴장이 여러 번 발생해도 프리킬 평가는 한 번만 예약합니다.
- 시작 시 중앙 메시지, 채팅, 최근 사건에 기록합니다.
- 1:1 비콘은 별도 설정으로 켤 수 있습니다. CBeam 엔티티를 사용하므로 기본값은 OFF입니다.

### 1:1 상황

- 살아 있는 죄수 1명과 살아 있는 간수 1명이 남으면 1:1 상황으로 기록합니다.
- 사망, 팀 변경, 퇴장 이벤트 후에만 평가해서 매초 풀스캔을 피합니다.
- 라운드 시작부터 1:1인 저인원 테스트 서버도 잡을 수 있게 라운드 시작 직후 한 번 평가합니다.
- 시작/종료 시 최근 사건에 기록하고 `css_jb status`에 상태를 표시합니다.
- `Beacon.Enabled`가 켜져 있으면 1:1 참가자 발밑에 CBeam 원형 비콘을 표시합니다.
- 비콘은 기본 OFF입니다. 켜더라도 기본 3초마다 기존 빔을 제거하고 새로 생성해 엔티티 누적을 피합니다.
- 죄수/간수 구분 없이 같은 다색 팔레트를 사용합니다.
- 기본 색상은 빨강, 노랑, 초록, 하늘, 보라 계열이며 `Beacon.Colors`에서 바꿀 수 있습니다.

### 고문시간 제거

고문시간은 정확한 서버 룰을 더 확인하기 전까지 현재 빌드에서 제거했습니다.

`!고문`, `!고문끝`, `css_jb torture` 계열 명령은 등록하지 않습니다. 죄수 1명 남음 자동 감지도 하지 않습니다.

### 최근 사건

최근 사건은 라운드 중 운영자가 상황을 되짚어 볼 수 있게 짧게 저장합니다.

현재 기록되는 사건:

- 누가 `!5` 카운트다운을 시작했는지
- 프리킬 시간이 시작/종료됐는지
- 누가 자동 반란자가 되었는지
- 관리자가 누굴 반란자로 지정/해제했는지

`css_jb incidents`, `css_jb 사건`, `css_jb 최근사건`으로 확인합니다. 라운드 시작과 맵 시작, 강제 초기화 때 기록은 비웁니다.

### 자유시간

- 전체 자유시간은 관리자 명령으로 시작/종료합니다.
- 개인 자유시간은 살아 있는 죄수에게 지정할 수 있습니다.
- 반란자가 되면 개인 자유시간은 제거됩니다.

### 반란자

- 죄수가 간수를 공격하면 반란자로 표시됩니다.
- 반란자는 빨간색 렌더 컬러를 적용합니다.
- 이미 빨간색이면 매초 `Render` 재기록을 건너뛰어 불필요한 네트워크 상태 변경을 줄입니다.
- 라운드 종료, 팀 변경, 퇴장, 초기화 때 상태를 정리합니다.

### 팀교환

- 죄수와 간수 사이에서만 상호 동의 팀교환을 요청할 수 있습니다.
- 요청자는 `css_jb team swap <대상>` 또는 `!교환 <대상>`을 사용합니다.
- 대상자는 `css_jb team accept`, `!수락`으로 수락하거나 `css_jb team deny`, `!거절`로 거절합니다.
- 요청은 기본 20초 뒤 만료됩니다.
- 한 번 거절된 요청자-대상 조합은 기본 60초 동안 다시 요청할 수 없습니다.
- 라운드 시작, 라운드 종료, 맵 변경, 초기화, 퇴장 때 요청 상태를 정리합니다.
- 팀밸 자유시간은 죽은 간수 포함 여부 등 서버 룰이 확정되지 않아 아직 구현하지 않습니다.

    ]

### 주요 설정

`PrisonersPerGuard`
: 간수 1명당 필요한 죄수 수입니다. 봇은 비율 계산에서 제외됩니다.

`AdminPermissions`
: 관리자 권한 목록입니다. 하나라도 보유하면 관리자 기능을 사용할 수 있습니다.

`DisableRadar`
: 플러그인 로드/맵 시작 때 레이더 비활성화 명령을 실행할지 여부입니다.

`DisableRadarCommand`
: 레이더 비활성화에 사용할 서버 명령입니다.

`BotsBlockLastRequest`
: `true`면 봇 죄수도 LR을 막습니다. 운영 기본값은 `true`입니다.

`GuardOrders.DefaultLocations`
: 간수 지시 장소 목록입니다. 메뉴에는 최대 4개가 표시되고 5번은 자유 입력입니다.

: 인간 죄수 몇 명당 인간 간수 1명을 허용할지 정합니다. 봇은 비율 계산에서 제외합니다.

`AdminPermissions`
: 관리자 명령 허용 권한입니다.

`CustomOrderInputTimeoutSeconds`
: 자유 입력 지시 입력 대기 시간입니다.

`DisableRadar`
: 플러그인 로드/맵 시작 때 레이더 비활성화 명령을 실행할지 여부입니다.

`DisableRadarCommand`
: 레이더 비활성화에 사용할 서버 명령입니다.

`AutoFreedayEnabled`
: 라운드 시작 때 전체 자유시간을 자동으로 시작할지 정합니다.

`AutoFreedayDurationSeconds`
: 자동 자유시간 유지 시간입니다. 기본값은 180초입니다.

`TeamSwapRequestTimeoutSeconds`
: 팀교환 요청 만료 시간입니다. 기본값은 20초입니다.

`TeamSwapDeclineCooldownSeconds`
: 팀교환 요청을 거절한 뒤 같은 요청자-대상 조합에 적용할 재요청 제한 시간입니다. 기본값은 60초입니다.

`Beacon`
: 1:1 비콘 설정입니다. `Enabled` 기본값은 `false`입니다. `Colors`는 `#RRGGBB` 형식의 공통 다색 팔레트입니다. 실서버 테스트 전에는 꺼두는 것을 권장합니다.

`GuardOrders`
: 간수 지시 메뉴, 시간 간격, 쿨다운, HUD, 기본 장소를 설정합니다.

`GuardOrders.NotificationSoundCooldownSeconds`
: 간수 지시 사운드 최소 간격입니다. 이후 사운드, 핑, 비콘도 같은 식으로 rate-limit를 적용합니다.
